# 00 · 概览

> 这一卷：讲整个游戏是什么、玩家一局在干什么、为什么这样设计。
> 如果只看一个文件，看这个。

## 一句话

玩家是一个**驱魔师**，在自己的办公室里挑选委托，进入闹鬼的场所，**用一台硬核可调参数的相机"拍下"鬼**，拍到关底 Boss 鬼即完成委托。

## 设计支柱

1. **相机即武器**。玩家的全部表达都压在相机参数上 —— ISO / 快门 / 光圈 / 对焦 / 测光，全部硬核、全部关联。调参就是战斗。
2. **鬼的"拍得到"是可设计的**。每只鬼的速度、亮度偏好、距离要求都是明确数值（见鬼谱），玩家必须推断出这一只需要什么组合的参数才能拍清。**拍鬼 = 解谜**。
3. **场景是生化危机式小关**。单关紧凑、封闭、有 Boss，一关打完回办公室。不是开放世界。
4. **办公室是成长线**。一开始相机功能受限，完成委托解锁更高 ISO、更广快门、新的镜头、鬼谱页、线索照。**循序渐进，不一次给满**。
5. **照片是真的**。照片是经过相机模拟管线输出的真 PNG（见 `10-camera-rules.md`），保存到磁盘，可以当成游戏成就 / 图鉴素材 / 案板证据。

## 核心循环

```
办公室（3D）
   │
   ├─ 看委托板 / 线索墙 / 图鉴 → 选择一个关卡
   │
   ▼
关卡（封闭场景）
   │
   ├─ 探索场景 + 氛围物（掉落物、声音、异象、碎片线索）
   ├─ 扰乱鬼出现 → 可选择拍 or 避开（拍到给图鉴 / 奖励，不强制）
   │
   ▼
遭遇关底 Boss 鬼
   │
   ├─ 必须调出正确相机参数组合
   └─ 拍下一张**达标**的 Boss 照（见判定规则）
   │
   ▼
回到办公室
   │
   ├─ Boss 照片上墙 / 图鉴更新 / 解锁下一章委托或新装备
   └─ 选下一关
```

## 玩家目标层级

| 层 | 目标 | 奖励 |
|---|---|---|
| 必做 | 拍到 Boss 达标照 | 通关、解锁进度 |
| 选做 | 拍到扰乱鬼 | 图鉴条目、线索碎片、可能的数值 buff |
| 隐藏 | 拍到罕见氛围物 / 特定时机异象 | 彩蛋、故事线延伸 |

## 拍照判定原则（摘要）

详细见 `10-camera-rules.md § 拍照判定`。核心规则：

- **必要条件**：Boss 出现在画面中央区域内。
- **画质阈值**：照片必须足够"清晰"——对焦在目标身上、主体运动模糊低于该鬼的阈值、曝光在可接受范围内、ISO 噪点未掩盖主体。
- **每只 Boss 有独立门槛**。简单的 Boss 宽松，后期 Boss 严苛。阈值写在鬼谱 Boss 条目里。
- 扰乱鬼判定更宽松 —— 只要"拍到一张能辨认的"就算。

## 不做什么（反设计）

- **不做战斗 / 不打鬼**。相机是唯一"武器"，拍到就是胜利。
- **不做大地图开放世界**。每关是手工关卡。
- **不做自动对焦即爽**。半按对焦是最低援助，**不提供实时曝光自动跟随**（代码里 auto-metering 有夹紧，是辅助参考不是自动模式）。
- **不做 QTE**。慢快门期间玩家可以移动，但移动产生相机位移模糊——由玩家自己决策是否站稳。
