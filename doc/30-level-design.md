# 30 · 关卡通用结构

> 这一卷：所有关卡共享的骨架。单个具体关卡写在 `60-levels/NN-slug.md`。
> 目标：任何一关都能按这张蓝图组装。

## 关卡的组成

每关由四层内容拼成：

| 层 | 作用 | 是否必需 |
|---|---|---|
| **地块** | 可走动的 3D 场景（生化危机式，封闭） | 必 |
| **Boss 鬼** | 唯一通关目标，必须拍到达标照 | 必 |
| **扰乱鬼** | 干扰、陪衬、叙事；可拍可不拍 | 0 – N |
| **氛围物 / 异象** | 道具、声音、掉落物、光影异变、碎片线索 | 必（氛围） |

## 关卡生命周期

```
进入（黑屏/开门转场）
   │
   ▼
前段：探索 + 拾取线索 + 首次遭遇扰乱鬼
   │
   ▼
中段：环境恶化（灯光、声音、异象密度提升）
   │         扰乱鬼类型 / 数量变化
   ▼
末段：Boss 触发条件满足（进入房间 / 拍到关键线索 / 倒计时）
   │
   ▼
Boss 遭遇战：玩家调参数 → 拍照 → 达标？
   │             ├─ 未达标 → Boss 逃跑/变换形态/再给一次机会（见下方"失败策略"）
   │             └─ 达标 → Boss 被"封存"在照片里
   ▼
退场：黑屏 → 办公室
```

## Boss 触发条件（选一种）

| 类型 | 说明 | 适用关 |
|---|---|---|
| 空间触发 | 走进关底房间 | 古宅、庙、戏台 |
| 线索触发 | 拍到 / 拾取 N 个特定物件 | 调查/解谜向 |
| 时间触发 | 关内经过 X 分钟 | 时限压迫感 |
| 仪式触发 | 按顺序交互 N 个道具（摆香烛、贴符等） | 民俗风格强化 |

## 失败策略

玩家拍照没达标**不要直接 GameOver**。选一种，关卡卡里写清楚：

| 策略 | 说明 |
|---|---|
| 重新显形 | Boss 消失 X 秒后重新出现，玩家继续尝试 |
| 形态切换 | Boss 换一种难拍的形态（速度变快 / 光变弱） |
| 惩罚扰乱 | 扰乱鬼数量提升，场景变暗 |
| 胶卷消耗 | 每次尝试消耗一张胶卷，用光则退出关卡（软失败） |

**推荐默认**：重新显形 + 每次失败扰乱加剧，三次后软失败退出。

## 氛围物预算（参考）

| 类别 | 一关数量 | 例子 |
|---|---|---|
| 掉落物 / 可拾取 | 3 – 6 | 日记页、符纸、旧照片 |
| 交互式道具 | 2 – 4 | 留声机、油灯、香炉 |
| 环境异象 | 5 – 8 | 风吹烛灭、镜中影像、自行滚动的物件、远处一闪即逝的身影 |
| 隐藏拍摄点 | 1 – 2 | 成就照：特定角度才能拍到的氛围构图 |

## 程序需要的关卡基础设施

- [ ] `LevelSceneController`：管理关卡状态机（进入/探索/Boss 遭遇/结算/退出）
- [ ] `GhostSpawner`：按时间 / 触发器生成扰乱鬼
- [ ] `BossTrigger`：四种触发条件的通用触发器
- [ ] `AmbientEvent`：一次性或随机的环境异象（风、烛、影子）
- [ ] `PhotoEvaluator`：拍照瞬间评估判定（见 `10-camera-rules.md § 拍照判定`）
- [ ] `ClueItem`：可拾取/可拍摄的线索物件
