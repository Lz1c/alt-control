## 2026-05-01 · v2 调味剂 — SCP 风诡异层补料

**触发**：v2 收紧后用户回看觉得纯治愈基调 30-45 分钟会乏味，登山路上需要少量"山的代价 / 不对劲"作为情绪反差。

**核心决定**：

- **加 1 处真扑面 jumpscare**，位置固定 = 段 4 风口 Boss A 首次半按对焦命中。形式：1-2 帧山君正脸闪现（眼神空洞、嘴张大、占屏 ~50%）+ 一记低吼音效 + 风暴粒子吃掉。**一次性，不重复**。
- **全片散布 SCP 风诡异事件**，7 类标签：被跟踪 / 错位 / 观察反转 / 计数错乱 / 时间感 / 声音错位 / 照片诡异。具体清单见 [`45-uncanny.md`](../45-uncanny.md) + [`60-levels/01-shan.md`](../60-levels/01-shan.md)。
- **三个硬边界**：
  - 山顶 Boss B 段**绝对干净**——情感终点不能毁
  - 全片只允许 **1** 处真扑面，其他都是积累式不安
  - 概率触发 + 不影响通关（治愈玩家可完全错过）
- **新增规则卷** [`doc/45-uncanny.md`](../45-uncanny.md)，作为诡异调味的护栏文件
- **段位分布**（与 45-uncanny 表同步）：trailhead 1 / 段 1 1 / 段 2 3 / 段 3 1 / 段 4 1+扑面 / 段 5 2 / 段 6 = 0
- **观察反转**条目（段 1：肉眼能瞄到、举相机就没了）刻意**反转 v2 核心机制**——强化"山记得太多"的不安。这是 SCP 式诡异最值得做的一条。
- **Steam 标签 / 美术宣传不打"恐怖"**——治愈是主调，诡异是隐藏味道
- **音效原则**：低频 / 风变形 / 远处人语 > 突发刺响。只有那 1 处扑面允许刺响

**反设计的对应放宽**：

- [`00-overview.md § 不做什么`](../00-overview.md)：放宽 jump scare 反设计，加调味剂段落
- [`40-ghost-rules.md § 治愈基调指引`](../40-ghost-rules.md)：第 3 条"不发出威胁声"加例外条款
- [`70-narrative.md § 不写什么`](../70-narrative.md)：放宽恐怖元素，加诡异调味段说明设计意图

放宽**仅**到 45-uncanny.md 定义的边界内，超出走 [`rules-revisions.md`](rules-revisions.md) 重新讨论。

**新机制需求**（W2-W3 实装）：

- `UncannyEvent` 基类 + `UncannyTrigger` 视野检测器
- `ObservationReverseObject`（观察反转专用，Layer 切换）
- `JumpscareTrigger`（仅风口，一次性）
- `AmbientStalker`（环境层散布）
- `FakePhotoInjector`（照片诡异，拍照存盘注入预制 PNG）
- `CrossSegmentAudio`（时间感，跨段位音源叠加）

**美术资产新增**（W4）：风口山君正脸 1 / 远处山君剪影 1 / 雾带静立人形 1 / 雾里掠过轮廓 1 / FakePhoto 3-5 / 主角自己剪影 1 / 雪地脚印贴花 1。

**未解决（已挪入 [`open-questions.md`](open-questions.md)）**：

- 扑面用 3D 模型还是 PNG 闪现
- 跟踪事件触发概率参数
- Steam 标签策略（"治愈 / 摄影 / 步行模拟" 是否够 / 要不要加 "atmospheric"）
- 一次性 vs 概率性 SCP 事件的具体配比

## 2026-05-01 · v2 收紧 — 单关登山治愈作业版

**触发**：交作业 deadline 一个月，v1 的体量（驱魔师 + 办公室 + 5 关 + 傩戏收集）做不完。用户拍板大幅收紧。

**核心决定**：

- **范围**：1 关 = 1 座山。所有相机功能塞进这一关。**没有关卡选择 / 没有办公室 hub / 没有面具墙 / 没有章节**。
- **题材**：第一人称登山摄影。视觉风照 [Cairn](https://store.steampowered.com/app/1588550/) — 漫画化电影插画，Mathieu Bablet 风。**直接借用 Cairn 的场景资产**（用法 = 交作业，不发布，资产合规问题不存在）。
- **核心机制不变**：肉眼看不见，相机能看见。但调性反转 —— v1 = 恐怖揭示鬼；v2 = **温柔显影山的记忆**。"鬼" rename 为 **"影 / 余像"**（技术 ID 保留 `ghost-*` 前缀避免破坏所有引用）。
- **叙事**：**代友登顶**。主角带着已故朋友的相机，沿朋友生前最后没走完的路上山，山顶替朋友拍下他想看的画面。
- **教学叙事化**：主角"不会用相机"——朋友是会用的那个。**教学 = 朋友留下的笔记本**，每段山路对应笔记本一页（"这里 2017 我拍过雾里残影，1.2s 长曝才显形"）。玩家学相机 = 主角破译朋友的笔记 = 替朋友走一遍。这把干巴巴 tutorial 全部消化成情感线。
- **不做攀岩**：第一人称纯走路。Cairn 的物理攀岩系统从资产里剥离 / 不接入。
- **保留实体相机外设决议**（2026-04-25）。开发期键盘替身全程可玩，硬件原型若 1 月内做不出不阻塞交付。
- **6 影骨架**：4 minion + 2 Boss，按相机功能反推：
  - 林祠静坐者（半按对焦 + 构图，破冰）
  - 雾里残影群（长曝消形）
  - 山鸦掠影（快门冻结）
  - **Boss A · 风口山君**（综合：快门 × ISO 噪点）
  - 星灵（长曝积累 / 星轨）
  - **Boss B · 山顶守者**（综合：景深 × 对焦 × 长曝）
- **通关条件**：登顶 + 拍下 Boss B（拍糊 = 中性结局"看到了但没拍下来"，拍达标 = 完整结局）。Boss A 全软，可拍可不拍。**PhotoEvaluator 出分但不卡通关** — 治愈基调 + 作业级体量不卡技能门槛。
- **新收集物 = 玩家相册**：拍的所有 PNG 自动存盘，山顶触发"主角翻相册"叙事段，把这一路真实拍到的照片当成情感载体。**这是替代面具墙的收集驱动**。

## 2026-04-25 · L1 升级为"快门主题关"（B+C 路线）

> ⚠️ 此条及以下所有 2026-04-25 决议被 2026-05-01 v2 收紧覆盖，保留作历史。

**问题**：删凝视档之后 L1 仍然只在教快门的一个属性——"冻结快速运动"。Boss 绕圈跑也单调。讨论后用户决定**把整个 L1 做深成"快门主题关"**，单关讲透快门的多种用法，而不是只教"拉快"。

**决定**：

- **Boss 战拆两阶段**：
  - **Phase 0 · 找鬼（长曝消形）**：玩家进操场时，跑道上飘着 5–8 个**夜训社团残影**（半透明循环跑者，无害不可收服）遮挡 Boss。Boss 静站中圈但被遮挡看不见。玩家必须**靠主席台栏杆触发"扶栏稳定"+ 长曝 ≥ 1s** → 残影被擦除 → Boss 武将面正脸显形 → 触发 Phase 1。
  - **Phase 1 · 拍鬼（多形态 + 摇镜）**：Boss 起步绕跑道（5 m/s 巡跑 + 9 m/s 冲刺）。**多形态显影**——同一鬼在不同快门下显示不同程度真相：
    - 站定 1/250 → 校服瘦高男（人形 ~60 糊弄过）
    - 站定 1/1000 → 面具边缘浮现（~70）
    - **摇镜 + 1/60** → 武将面 + 盔甲虚影 + 速度线背景（~90 + 隐藏成就）
- **快门范围扩展**：原 1/30–1/1000 → **1s–1/1000**（含 1s/1/2/1/4/1/8/1/15 慢档位）。慢档为长曝必需，快档为冻结必需。
- **扰乱鬼扩成 5 只**，每只教 Boss 战需要的一招：
  - 长椅鬼（基础操作 / 破冰）
  - **粉笔字鬼**（新加，教学楼黑板，长曝积累）
  - 白衣女（楼梯，路径强制横向，摇镜法预热）
  - 冲跑小孩（操场入口，冻结）
  - **夜训社团残影群**（新加，操场内，长曝消形 + Boss Phase 0 启动器）
- **隐藏成就 "快眼" → "追风"**：摇镜 + 1/60 拍中达标，奖励真用 panning 技术的玩家。
- **新机制需求**（Unity 实装阶段加，PhotoEvaluator 要扩维度）：
  - 长曝防抖（扶栏 buff / 角速度阈值）
  - 摇镜评分（相机角速度匹配 + 背景流线检测）
  - 多形态显影（按快门档位插值显示不同形态层）
  - 长曝 alpha 衰减（持续移动主体随曝光时间趋于 0）
- **关卡时长预估**：从 20 分钟 → **25–30 分钟**（5 只扰乱鬼 + 两阶段 Boss）。

**配比哲学**：3 关游戏（暂定）每关都是"小型完整体验"，L1 不能只教一个概念。但 L2/L3 题材未定，所以 L1 单独做深，**不预设后面**——roadmap 留空。

**未解决**：武将面具 vs 校园学生的调性问题仍开放（A 留武将吃违和感 / B 换体系 / C 改鬼定位），不影响本轮设计推进。

**影响文件**：

- `doc/41-ghosts/ghost-yundong-yuan.md`（行为段拆 Phase 0/1，机械数值表大改，拍照判定加多形态/摇镜两节，关联谜题段重写）
- `doc/plan/knowledge-lock-chain.md`（L1 节大改：教学目标 / 鬼对应 / 快门范围 / 修改记录）

## 2026-04-25 · L1 Boss 删凝视档 —— 走"狠路"堵教学漏洞

**问题**：原 Boss 三档（凝视 0 / 巡跑 5 / 冲刺 9 m/s）允许玩家"等凝视档拍中拿 60 通关"，**整个 L1 不需要把快门拉过 1/60 也能过**——快门优先教学曲线作废。

**决定**：

- **删凝视档**。Boss 只剩巡跑（5 m/s）+ 冲刺（9 m/s）。**无停止状态**，玩家必须真把快门拉到 1/250+ 才能通关。
- **哨音改挂冲刺档预告**：冲刺起步前 0.3 秒，脚步突然变重 + 一声哨"嗶——" → 给玩家时间举相机切 1/1000。叙事上仍是"运动场口哨"，合理。
- **取景器"鬼头变清晰"机制随凝视档一起删除**。"相机能看穿"由"肉眼完全看不见鬼"这条基本机制承载，不需要额外信号。
- **passingScore 仍 60**。难度地板已经抬了（1/250 是新下限），分数门槛再涨过狠。
- **隐藏成就 ★★★ 替换**：原"读心术（凝视瞬间拍中）" → "**快眼**（在冲刺档那 3 秒内拍中达标）"。同样是高难时机判定，奖励玩家真去碰冲刺档。
- **白衣女路径同步打补丁**（待 `/new-ghost minion 白衣女` 落卡时落实）：飘动方向**强制横向**（楼梯转角横向飘过），堵掉"径向飘动→视觉位移≈0→慢快门也清晰"的漏洞。
- **软失败 / 无限胶卷 / 不追击 全保留**——狠在机制，软在惩罚。

**配比哲学**：L1 教学硬，挫败软。机制门槛抬到 1/250，但拍糊 0 代价、随时重来——避免新手劝退，又不让玩家"装睡过课"。

**影响文件**：`doc/41-ghosts/ghost-yundong-yuan.md`（行为 + 数值表 + 打法表 + 拍照判定 + 收服反馈）/ `doc/plan/knowledge-lock-chain.md`（Boss 行）/ 桌面 `L1-校园之夜-设计概要.md` 同步。

## 2026-04-25 · 项目交互形态定稿 —— 实体相机外设 + 双屏混合输入

**这是项目级核心决定**：

- **硬件形态**：游戏的相机是**实体物理外设**（改装相机硬件，连 PC），不是纯键鼠虚拟相机。玩家**真的举起一台相机**对着屏幕拍。
- **双屏布局**：
  - **PC 屏幕** = 第一人称视角（玩家走路看的是 PC 屏；不举相机时只能看场景）
  - **相机屏幕** = 取景器画面（举起相机时才亮，显示 PC 渲染传过来的取景画面 + HUD）
- **混合输入**：
  - **WASD（键盘）** = 移动 + 转视角（PC 屏视角控制）
  - **实体相机** = 举起 / 调参 / 拍照 / 回看
- **举起检测方案**：陀螺仪 pitch 角（抬到水平 60° 自动亮屏，垂下 30° 持续 0.8 秒自动暗屏）；可选加距离传感器加强。**不用按钮开关**（破坏代入感）。
- **L1 玩家用到的物理控件**（4 个）：
  - 快门按钮（两段式：半按对焦+测光，全按拍照）
  - 主转盘 → 调快门
  - 副转盘 → 调 ISO
  - 回看按钮 ▶
- **后期才解锁**（L1 物理上存在但不响应）：模式转盘、EV+/-、AF/MF、对焦环、变焦环、WB、闪光、菜单/删除、录像键
- **取景器 UI**：取景画面 + 中央对焦框（半按变绿）+ 测光指针 + 底部 HUD（S/ISO/f）+ 回看模式（拍后照片大图 + 成败原因文字"主体模糊/偏离中心/曝光不足"，**不显示分数**）

**硬件原型暂时挂起**（屏幕尺寸 / 传输协议 / 原型阶段都未定），**软件先做**。开发期用键盘替身：
- O = 半按 / P = 全按 / 滚轮 = 主转盘 / ALT+滚轮 = 副转盘 / Tab = 回看 / Q = 切换举起状态

软件层抽象出"5 路相机输入"接口，硬件原型出来后只换 HID 实现，不改游戏逻辑。

## 2026-04-25 · L1 细化决定（level-planner 5-block 输出后的 Q12-Q21）

接 level-planner 盘点输出，用户确认"先按推荐"。所有决定一次清空：

- **Q12 PhotoEvaluator 权重**：直接采用 `10-camera-rules.md` 给的基线 —— 对焦 25 / blur 30 / 曝光 20 / 噪点 15 / 占屏 10。L1 用基线，未来关卡发现问题再调。
- **Q13 5 段路径过渡机制**：触发器 + 软引导异象（飘灯 / 异象把玩家引向下一段，路只有一条）。不做钥匙、不做关键道具。
- **Q14 校园氛围异象清单**：6 件全收：
  - 校门：保安亭电视雪花屏 + 自动门反复开关 + 保安亭里旧学生证（剧情暗示运动员鬼身份）
  - 走廊：教室门突然自己关 + 黑板上粉笔自动写字写一半停
  - 楼梯：上方传来跳绳/踏步声但无人
  - 操场入口：体育馆窗里身影一闪
  - 操场：广播突响"全体跑步——走！" + 跑道某段灯灭再亮
- **Q15 操场环境光**：4 盏跑道角灯（4000K 冷白，EV 4-5）+ 主席台聚光（3000K 暖黄，EV 6-7）+ 月光基底（EV 2）。物理上鼓励"等 Boss 走到亮处再拍"。
- **Q16 Boss 凝视回头窗口提示**：哨音提示（脚步停 + 一声哨音）+ 弱化的视觉提示（取景器里那一刻鬼头变清晰一点）。哨音叙事合理；视觉强化"相机能看穿"核心机制。

## 2026-04-25 · 核心机制定稿 —— 双层可见性 + 收服掉面具

- **双层可见性**：肉眼看不见鬼；只有透过相机取景器才能看到。这是项目级核心机制，**不可回退**。
- **鬼的"未收服"形态**：鬼头本身就是"类傩戏面具"的形貌（与对应的傩戏面具高度相似但能看出是活体鬼头，有细微生物特征区别），不是肉身戴面具。
- **收服 = 拍出达标照片**。Boss 拍中后，鬼直接凝固成实体傩戏面具掉落到场景里，玩家走过去拾取。
- **面具是收集进度的物理载体**。拾取的面具回办公室挂面具墙，墙就是图鉴的实体形态。每只 Boss 对应一张**确定的**面具（一只鬼掉一张）。
- **设计含义**：
  - 现代/古代任意题材都能容纳傩戏面具（解决了"现代校园 + 木雕面具"的违和）
  - 玩家从相机取景器里看到的鬼头轮廓 = 提前预告会掉什么面具 → 收集驱动力
  - 没相机时玩家在闹鬼场景里其实是"完全瞎"的，营造氛围张力
- **影响范围**：00-overview / 20-office-hub / 50-mask-rules / CLAUDE.md 同步更新；不影响 10-camera-rules（相机参数规则不变）。
