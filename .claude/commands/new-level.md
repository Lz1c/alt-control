## v2 上下文（必读）

v2 项目是**单关登山治愈版**（详见 [`doc/00-overview.md`](doc/00-overview.md)）。唯一关已建：[`doc/60-levels/01-shan.md`](doc/60-levels/01-shan.md)。一个月交作业 deadline 下**不应该再建第二关**。

## 第一步 · 警告 + 确认

**在执行任何动作之前**，向用户回报：

```
⚠️ v2 项目是单关登山版，唯一关已建（doc/60-levels/01-shan.md）。
   再建一关意味着项目范围扩张，会冲击 4 周交付时间表。

确认要继续吗？如果是，请告诉我：
- 这一关与 01-shan 的关系（独立 / 续集 / 平行 / 替换 01？）
- 项目时间表是否同步调整
- 是否要在 doc/plan/rules-revisions.md 先起一条范围扩张草稿
```

**等用户明确说"继续 / 我确认"再往下做。** 用户犹豫或问"为什么"就停手并解释。

## 第二步 · 执行（仅在用户明确确认后）

解析参数：
- 第一个词 = 文件 slug（英文/拼音，小写，中划线；如 `shan-bei`）
- 第二个词 = 关名（中文）

如果用户没给全，先问。

1. **读 [`doc/30-level-design.md § 关卡条目模板`](doc/30-level-design.md)**，整段复制作为起点。
2. **扫 [`doc/60-levels/`](doc/60-levels/)**，取下一个编号（`02-`、`03-` … 不要和现有重复）。
3. **读 [`doc/41-ghosts/README.md`](doc/41-ghosts/README.md)**，列出 v2 已有的 6 张影卡给用户选作为本关用，或允许新建（后续用 `/new-ghost`）。
4. **问用户最少的必要信息**（一次问一条）：
   - 关底 Boss 影 ID（已有 / 新建）
   - 场景地点（v2 调性：山系 / 自然 / 治愈）
   - 通关条件
5. **按 v2 单关模板填好，未填项用 `TBD` 占位**（不要编故事 / 不要写 v1 民俗 / 不要写恐怖元素）。
6. **新建文件** `doc/60-levels/NN-<slug>.md`。
7. **更新 [`doc/60-levels/README.md`](doc/60-levels/README.md) 当前条目表**，添加这一行。
8. **回报**：文件路径、编号、关底 Boss ID、提示用户同步更新 `doc/plan/roadmap.md`（项目时间表会受冲击）。

## 注意

- ⚠️ **不要写**：v1 题材（古宅 / 戏班 / 渔村 / 校园 / 民俗）/ 多种 Boss 触发类型（v2 是线性段位推进，不是空间触发）/ 失败策略选项（v2 治愈基调统一软克制）
- ⚠️ `doc/51-masks/` 已物理删除，v2 没有面具系统，不要试图新建该目录
- 用户给的关底影如果不在 v2 鬼谱里，**不要自动调用 `/new-ghost`**，只在回报里建议手动跑，避免命令嵌套
- 如果整个流程让你觉得"这是 v1 思维"，立刻停手并向用户回报，让用户重新决定
