用户输入：`$ARGUMENTS`

解析参数：
- 第一个词 = 文件 slug（英文/拼音，小写，中划线；如 `gu-xiang`）
- 第二个词 = 关名（中文；如 `古巷夜啼`）
- 第三个词（可选）= 章节号（如 `序章`、`第一章`；没给就设 "TBD"）

如果用户没给全，先问。

## 你要做的事

1. **读 `doc/30-level-design.md` § 关卡条目模板**，整段复制作为起点。
2. **扫 `doc/60-levels/`**，取下一个编号（`01-`、`02-`… —— 不要和现有重复）。
3. **读 `doc/40-ghost-bestiary.md`**，列出已有的 Boss 鬼 ID 给用户选作为关底，或者允许用户输入一个新的（后续用 `/new-ghost` 填）。
4. **问用户最少的必要信息**（一次问一条）：
   - 关底 Boss 鬼 ID（已有 / 新建）
   - 场景地点一句话（古宅 / 戏班旧院 / 山村 / 渔村古祠堂 …）
   - 日夜 / 氛围（白 / 黄昏 / 夜 / 室内）
   - Boss 触发类型（空间 / 线索 / 时间 / 仪式 —— 见 `30-level-design.md` 的四种）
5. **按模板填好，未填项用 `TBD` 占位**（不要编故事）。
6. **新建文件** `doc/60-levels/NN-<slug>.md`（NN 是前一步取到的编号）。
7. **更新 `doc/60-levels/README.md` 的 "列表（占位）" 表**，添加这一行。
8. **回报**：文件路径、编号、关底 Boss ID、你觉得下一步要补什么（例如"扰乱鬼还没定"）。

## 注意

- `NN` 用两位数字前缀（`01`、`02`、`11`、`12`）；同章节相邻编号，跨章节跳号（第一章从 `11` 起）。
- 如果章节号是 "序章"，用 `01` 起；"第一章" 用 `11` 起；"第二章" 用 `21` 起。依此类推。
- 用户给的关底 Boss 鬼如果鬼谱里不存在，**不要自动调用 `/new-ghost`**，只在回报里建议他手动跑 `/new-ghost`，避免两个命令互相嵌套造成意外。
