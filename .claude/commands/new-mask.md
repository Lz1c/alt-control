用户输入：`$ARGUMENTS`

解析参数：第一个词 = 体系 slug（dejiang / cuotaiji / dixi / chenzhou / maonan / zhuang / guanzuo / ...），其余 = 面具显示名。如果用户没给，先问。

## 你要做的事

1. **读 `doc/50-mask-rules.md`**，找到 "面具卡模板" 整段。
2. **读 `doc/51-masks/README.md`**，扫现有面具 ID 取下一个序号。注意：若该 `mask-<体系>-<NN>` 已经以"未设计"占位（由之前的 `/new-ghost` 添加），**直接复用这个 ID**，不要另起号。
3. **读 `doc/90-reference/nuo-masks.md` §三中对应体系的段落**，把典型造型特征、材质、主色做成输入。
4. **问用户最少的必要信息**：
   - 仪式语境（驱疫 / 祭祖 / 还愿 / 祈子 / 丧葬超度 / 其他）
   - 这只面具在游戏里要做"什么样的鬼"的面具 —— 一句话（比如"一个幼年夭折、戴银饰的少女"）
5. **根据体系特点自动填**：
   - 主色（五色体系）
   - 材质（参考调研：木雕 + 漆 / 纸扎 + 布 / 盔面连体 / 活口机关 / 纸胎漆布 …）
   - 造型元素（獠牙 / 三眼 / 鼓目 / 歪嘴 / 吐舌 / 犄角 / 额心瞳 里选 1–2 项）
6. **原创变形**字段必填，**绝对不能**写"和德江某面具一模一样" —— 必须是原创改编（不对称、材质异化、加工艺、减细节等任选一条）。
7. **新建文件 `doc/51-masks/<mask-id>.md`**（用 Write 工具）。模板的 `###` 调成 `#`（文件级 heading）。
8. **更新 `doc/51-masks/README.md` 的条目表**：若该 ID 是占位行，把"状态"列改为"已设计"并补关联鬼；若是全新 ID，追加一行。
9. **回报**：面具 ID、体系、主色三行摘要。

## 注意

- 如果体系是"藏羌姆 / 藏戏"，先确认用户知道 `doc/50-mask-rules.md § 三大风险` 第 3 条（宗教冒犯风险）再写。
- **美术参考**字段先写 "TBD"，等美术真的画了再回填。
- 扰乱鬼的面具若是"无面具"（例如影子），不走本命令，手动改鬼卡即可。
